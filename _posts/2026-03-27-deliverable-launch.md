Deliverable D.2.1.1 is a strong candidate for a continuous publication model. The project documentation already describes a benchmarking process that evolves over time as new models and datasets become available, so a public GitHub Pages site is a better operational format than a single frozen report.

The current site structure does four things:

- turns the deliverable into a maintainable public-facing web resource
- separates stable project context from fast-moving benchmark updates
- publishes the strongest current evidence first, rather than waiting for every technology track to mature
- makes later additions straightforward as new comparison tracks are ready

At the time of initialization, the most complete material in the workspace covers:

- Named Entity Recognition
- PII detection and anonymization
- the shared methodology and evaluation logic behind those comparisons

Other planned technology areas remain visible on the site as future tracks, but they are not presented as completed comparisons before the evidence exists.
